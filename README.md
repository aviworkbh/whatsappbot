# בוט וואטסאפ - תזמון הודעות לפי תאריך שמוזכר בקבוצה

בוט זה מתחבר לוואטסאפ שלכם (כמו מכשיר נוסף מקושר), מאזין להודעות בקבוצות,
מזהה תאריכים בתוך תוכן ההודעה (בעברית או באנגלית), ומתזמן שליחה חוזרת
של אותה הודעה לאותה קבוצה במועד שזוהה.

## חשוב לדעת לפני שמתחילים

- הבוט משתמש בספריית `whatsapp-web.js`, שהיא ספרייה **לא רשמית** שמדמה חיבור
  דרך WhatsApp Web (Puppeteer/Chromium). זו הסיבה שהיא יכולה לקרוא הודעות
  מקבוצות (בניגוד ל-API הרשמי של WhatsApp Business, שלא תומך בכך).
- מכיוון שזו ספרייה לא רשמית, יש סיכון תיאורטי (נמוך אך קיים) לחסימת המספר
  ע"י וואטסאפ, בעיקר אם שולחים כמויות גדולות של הודעות אוטומטיות. מומלץ
  להשתמש במספר ייעודי לבוט ולא במספר האישי שלכם.
- הבוט צריך להישאר "דלוק" (online) על השרת כל הזמן כדי להאזין להודעות
  ולשלוח בזמן - למשל בעזרת PM2 או שירות systemd (ראו בהמשך).

## התקנה

1. ודאו שמותקן Node.js גרסה 18 ומעלה בשרת.
2. התקינו את ה-dependencies:

   ```bash
   npm install
   ```

3. בשרתי לינוקס יתכן שתצטרכו להתקין תלויות מערכת עבור Chromium
   (ש-Puppeteer מוריד אוטומטית). לדוגמה באובונטו/דביאן:

   ```bash
   sudo apt-get update && sudo apt-get install -y \
     ca-certificates fonts-liberation libasound2 libatk-bridge2.0-0 \
     libatk1.0-0 libcups2 libdbus-1-3 libdrm2 libgbm1 libgtk-3-0 \
     libnspr4 libnss3 libx11-xcb1 libxcomposite1 libxdamage1 \
     libxrandr2 xdg-utils libxshmfence1
   ```

## הרצה

```bash
npm start
```

בהרצה הראשונה יופיע קוד QR בטרמינל. סרקו אותו מהטלפון:
**וואטסאפ > הגדרות > מכשירים מקושרים > קישור מכשיר**.

לאחר הסריקה החיבור נשמר בתיקיית `session/`, כך שלא תצטרכו לסרוק שוב
בכל הפעלה מחדש (כל עוד לא מוחקים את התיקייה או מתנתקים מהטלפון).

## איך זה עובד

1. הבוט מאזין לכל ההודעות הנכנסות מקבוצות (`chat.isGroup === true`).
2. עבור כל הודעה, `dateParser.js` מנסה לזהות תאריך/ים בטקסט. הפרסר נכתב
   באופן ייעודי לעברית (לא מסתמך על תמיכה בעברית בספריית `chrono-node`,
   כי כזו תמיכה **אינה קיימת** בספרייה) ותומך בין השאר ב:
   - תאריך מספרי, עם או בלי שנה: `08/07`, `08/07/2027`, `15.8.2026`
     (אם לא צוינה שנה - מניחים את השנה הנוכחית, ואם התאריך כבר עבר -
     עוברים אוטומטית לשנה הבאה)
   - שם חודש בעברית: "20 באוגוסט", "ב-20 לאוגוסט", "8 ליולי"
   - סמיכות מספר-סידורי: "20 לשביעי" (=20 בחודש ה-7)
   - מילים יחסיות: "מחר", "מחרתיים", "היום"
   - תאריכים באנגלית (כגיבוי, דרך `chrono-node`)
3. אם נמצא תאריך עתידי - הבוט שולח **מיד** הודעת אישור לאותה קבוצה בנוסח:
   ```
   ההודעה התקבלה לתאריך 16/07/2026 בשעה 14:32.
   ההודעה תוקפץ שוב בתאריך 15/08/2026 בשעה 08:00.
   ```
   ובמקביל נשמרת רשומה בקובץ `data/scheduled.json` ומתוזמן Job עם
   `node-schedule` שישלח את **אותה הודעה המקורית בדיוק** בחזרה לאותה
   קבוצה, במועד שזוהה (אם לא צוינה שעה בהודעה - ברירת המחדל היא 08:00,
   ניתן לשנות ב-`DEFAULT_HOUR` בקובץ `dateParser.js`).
4. אם השרת מופעל מחדש, `rescheduleAll()` טוענת את כל התזמונים
   הפתוחים מהקובץ ומתזמנת אותם מחדש - כלום לא "נשכח".
5. לאחר השליחה, הרשומה נמחקת מהקובץ.

## הגבלת הבוט לקבוצות מסוימות בלבד

בקובץ `index.js`, מלאו את המערך:

```js
const ALLOWED_GROUP_NAMES = ['שם הקבוצה שלי', 'קבוצה נוספת'];
```

אם המערך ריק, הבוט יפעל על כל קבוצה שהוא חבר בה.

## מצב פיתוח - הפעלה מחדש אוטומטית עם nodemon

בזמן שאתם עורכים קוד ורוצים שהבוט יתחיל מחדש אוטומטית בכל שמירה
(בלי להריץ `npm start` ידנית בכל פעם), אפשר להשתמש ב-`nodemon`.

```bash
npm install       # יתקין גם את nodemon (הוגדר כ-devDependency)
npm run dev
```

**חשוב לדעת:**
- **אין צורך לסרוק QR מחדש** - תיקיית `session/` נשמרת על הדיסק בין הרצות,
  כך שההתחברות נשמרת גם כשהתהליך מופעל מחדש.
- קובץ `nodemon.json` מגדיר במפורש לעקוב רק אחרי `index.js`, `dateParser.js`
  ו-`storage.js` - **ולא** אחרי תיקיית `data/`. זה קריטי: הבוט עצמו כותב
  ל-`data/scheduled.json` בכל פעם שהוא מתזמן הודעה, ואם `nodemon` היה עוקב
  גם אחרי תיקייה זו, זה היה יוצר לולאת הפעלות-מחדש אינסופית (כתיבה לקובץ ->
  restart -> כתיבה לקובץ -> restart...).
- הגדרנו ל-`nodemon` לשלוח אות `SIGINT` בכל restart, כך ש-Chromium נסגר
  בצורה מסודרת ולא נשארים תהליכים "רפאים" ברקע.
- **למצב ייצור (production) בשרת - השתמשו ב-`npm start` הרגיל או ב-Docker/PM2,
  לא ב-`nodemon`**. `nodemon` מיועד לפיתוח בלבד.

## שלב 1: הרצה מקומית על המחשב (לבדיקות)

לפני העלאה לשרת, מומלץ לבדוק שהכל עובד מקומית. הקוד זהה לחלוטין -
פשוט מריצים אותו ישירות בלי Docker:

1. ודאו ש-Node.js גרסה 18 ומעלה מותקן אצלכם (`node -v`). אם לא -
   הורידו מ-https://nodejs.org (מומלץ גרסת LTS).
2. בטרמינל, מהתיקייה הראשית של הפרויקט:

   ```bash
   npm install
   npm start
   ```

   בהרצה הראשונה `npm install` יוריד גם Chromium אוטומטית (חלק מ-Puppeteer,
   שעליו מבוססת whatsapp-web.js) - אין צורך להתקין Chrome בעצמכם.

3. יופיע קוד QR בטרמינל. סרקו אותו מהטלפון:
   וואטסאפ > הגדרות > מכשירים מקושרים > קישור מכשיר.
4. שלחו הודעה עם תאריך לקבוצת בדיקה ווודאו בלוגים שהתאריך זוהה
   ("זוהה תאריך... ") ושבמועד שנקבע נשלחת ההודעה חזרה.

**שימו לב:** הרצה כך תלויה בכך שהמחשב דלוק והטרמינל פתוח. ברגע שסוגרים
את הטרמינל (או Ctrl+C) הבוט נעצר. זה בסדר גמור לבדיקות - לריצה קבועה
24/7 עוברים לשרת (שלב 2 למטה).

כדי לעצור את הבוט: `Ctrl+C` בטרמינל. כדי להתחיל מחיבור נקי (למשל אם
משהו נתקע), מחקו את תיקיית `session/` וסרקו QR מחדש.

## שלב 2: העלאה לשרת עם Docker (מומלץ)

בפרויקט מצורפים `Dockerfile` ו-`docker-compose.yml` מוכנים, כולל כל תלויות
ה-Chromium הנדרשות, ו-volumes ששומרים את תיקיות `session/` ו-`data/`
מחוץ לקונטיינר - כך שגם אם מפילים/מעדכנים את הקונטיינר, **לא תצטרכו לסרוק QR מחדש**
וההודעות המתוזמנות לא יאבדו.

```bash
docker compose up -d --build
docker compose logs -f      # לצפייה בקוד ה-QR ובלוגים
```

לעצירה:

```bash
docker compose down
```

לעדכון הקוד (למשל אחרי שינוי בקובץ index.js):

```bash
docker compose up -d --build
```

## הרצה מתמשכת על שרת בלי Docker (חלופה: PM2)

```bash
npm install -g pm2
pm2 start index.js --name whatsapp-date-bot
pm2 save
pm2 startup   # מגדיר הפעלה אוטומטית גם אחרי ריסטארט לשרת
```

צפייה בלוגים:

```bash
pm2 logs whatsapp-date-bot
```

## מבנה הפרויקט

```
whatsapp-date-bot/
├── index.js         # הלוגיקה הראשית של הבוט
├── dateParser.js     # זיהוי תאריכים בתוך טקסט (עברית/אנגלית)
├── storage.js         # שמירה/טעינה של הודעות מתוזמנות מקובץ JSON
├── package.json
├── data/
│   └── scheduled.json # נוצר אוטומטית - רשימת ההודעות הממתינות לשליחה
└── session/            # נוצר אוטומטית - נתוני החיבור לוואטסאפ (לא לשתף!)
```

## התאמות אפשריות

- **שעת שליחה ברירת מחדל**: שנו `DEFAULT_HOUR` / `DEFAULT_MINUTE` בקובץ `dateParser.js`.
- **שינוי טקסט ההודעה החוזרת** (למשל להוסיף "תזכורת:" בתחילת ההודעה):
  ערכו את `sendScheduledMessage` ב-`index.js`, לדוגמה:
  ```js
  await client.sendMessage(item.chatId, `תזכורת:\n${item.text}`);
  ```
- **מספר תאריכים בהודעה אחת**: הקוד תומך בזיהוי כמה תאריכים באותה הודעה
  ומתזמן שליחה נפרדת לכל אחד מהם.
